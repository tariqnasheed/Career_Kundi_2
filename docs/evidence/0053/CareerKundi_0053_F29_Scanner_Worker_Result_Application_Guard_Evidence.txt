7. Exact implementation behaviour
---------------------------------
Module: backend/app/platform/evidence/attachment_scan_worker_result_application.py
Entrypoint: apply_attachment_scan_worker_result(db, owner_user_id, scan_job_id,
            expected_content_hash_snapshot, plan: ScanJobUpdatePlan)

Decisions:
  applied | already_applied | not_found | evidence_not_found | hash_mismatch |
  not_reserved | action_not_allowed | plan_not_persistable | conflicting_replay

State surface:
  reserved -> completed
  reserved -> failed
Allowed actions: COMPLETE_PASSED, COMPLETE_FAILED, QUARANTINE_REQUIRED, MARK_ERROR
QUARANTINE_REQUIRED: job_status=completed, safety=scan_failed (never quarantined; no file move)
Rejects: CANCEL_JOB, RESERVE_JOB, NO_OP, apply_to_database=False, cancelled target, queued source

F22 reuse:
  normalize_scan_job_update_plan
  assert_scan_job_update_allowed
  apply_normalized_scan_job_update_to_loaded_job (new shared helper; no fetch/commit)
Public apply_scan_job_update_plan unchanged for non-worker callers (delegates to helper).

8. Lock and transaction proof
-----------------------------
Lock order: AttachmentScanJob FOR UPDATE, then EvidenceRecord FOR UPDATE
One transaction: lock → validate → mutate job only → commit once
Rejected / soft-replay paths: rollback; no field writes on already_applied
Does not call public unlocked apply_scan_job_update_plan as outer F29 path
Tests prove concurrent identical double-apply (APPLIED + ALREADY_APPLIED) and
concurrent conflicting double-apply (APPLIED + CONFLICTING_REPLAY) via two sessions
+ asyncio.Barrier.

9. Triple-hash proof
--------------------
EvidenceRecord.content_hash == AttachmentScanJob.content_hash_snapshot
  == expected_content_hash_snapshot
Covered by tests: expected mismatch, live hash drift, aligned success,
missing evidence, other-owner evidence → evidence_not_found.
No file/storage open.

10. Idempotency proof
---------------------
Six-field effective projection after F22 non-None semantics:
  job_status, attachment_safety_status, engine_name, engine_version,
  safe_error_code, safe_error_message
Exact match → already_applied; completed_at/updated_at/attempt_count unchanged
Mismatch on safe_error_message or engine → conflicting_replay; no mutation
